# M0.3 — Odysseus UX-fidelity checklist (SPEC §13 O12)

> The mechanical DoD for chunk M0.3, extracted from `ODYSSEUS-REF/` source
> (static/js/*) before building. "Looks and behaves like Odysseus" means the
> boxes below. We clone the **interaction model**, re-implemented typesafe on
> the putty-ai-design kit — not the code, and deliberately **not every flourish**
> (the out-of-scope list at the bottom is a decision, not an omission).
>
> Source-of-truth files studied: `sidebar-layout.js`, `section-management.js`,
> `init.js`, `windowDrag.js`, `windowResize.js`, `modalSnap.js`,
> `tileManager.js`, `toolWindowZOrder.js`, `modalManager.js`, `ui.js`,
> `search-chat.js`, `keyboard-shortcuts.js`, `slashCommands.js`,
> `slashAutocomplete.js`, `theme.js`, `spinner.js`.

## A. Shell layout

- [x] Sidebar (default ~340px) + main area; sidebar sections in fixed order:
      Chats · Models · Tools (PuttyU surfaces replace Odysseus's), user bar at
      the bottom.
- [x] Hamburger toggles full sidebar ↔ **icon rail** (thin strip of section
      icons); clicking a rail icon expands the sidebar.
- [x] Sidebar **resizable by drag handle** (min 200 / max 700; drag below ~150
      collapses to rail); width persisted (localStorage) and published as a CSS
      var so window math stays accurate.
- [x] Sections **collapse/expand** on header click (chevron), persisted per
      section id.
- [x] Topbar lives inside the chat container (title + meta), not a global bar.

## B. Dockable window manager (tool windows)

- [x] Tool windows are floating panels: **inline fixed-px positioning**
      (left/top/width/height), not transforms — snap/dock math reads rects and
      writes px.
- [x] **Drag** by header (4px move threshold; click-swallow after a real drag
      so header clicks don't fire).
- [x] **Resize** by border proximity (~7px edge zones, corner diagonals), no
      visible handles; min 320×200; clamped to viewport; size persisted per
      window id.
- [x] **Edge dock** left/right (~60px snap zone, ~80px undock): docked window
      becomes full-height; body reserves room via a CSS var; dock width
      user-resizable + persisted; undock restores the exact pre-dock rect.
- [x] Docking that would squeeze the chat under ~380px **auto-collapses the
      sidebar to the rail** (and restores it on undock).
- [x] **Tile zones** while dragging: top strip = fullscreen/maximize, side
      edges = half-screen tiles; translucent ghost previews the zone; springy
      snap easing (`cubic-bezier(0.34,1.56,0.64,1)`, ~0.22s); dragging a
      snapped window unsnaps it; tiles re-clamp on viewport/sidebar changes.
- [x] **Z-order:** most-recently-opened/interacted window is on top, always.
- [x] **Minimize to dock chips**: minimize button next to close; chips in a
      bottom dock restore on click; sidebar/rail click on a minimized tool
      restores instead of re-opening.

## C. Cmd/Ctrl-K palette

- [x] `Ctrl/Cmd-K` **toggles** the overlay. Odysseus scopes it to
      conversation search; PuttyU extends it to global search + navigation
      (SPEC F11) — same interaction shell.
- [x] Debounced (~300ms) query; results **grouped** (by session/surface) with
      highlighted match snippets + relative timestamps.
- [x] ArrowUp/Down move selection (scrolls into view), Enter executes
      (navigates), Escape/outside-click closes. Empty query = empty state.

## D. Slash commands (composer)

- [x] Popup only when input **starts with `/`** and has no newline; anchored
      above the textarea (below if no room); role=listbox.
- [x] Rows grouped by category; each row = token + help text; **prefix match
      beats substring**; exact parent command surfaces its subcommands.
- [x] ArrowUp/Down wrap; **Tab always inserts** the token; Enter inserts
      unless a complete command was typed (then submits); Escape hides;
      mousedown inserts (preventDefault so blur doesn't race).
- [x] Dispatch on submit: known command executes (echoed as a
      non-LLM-context bubble); unknown command → Levenshtein "did you mean …?"
      (≤2 edits), never silently sent to the model.
- [x] v1 command set (PuttyU): `/new`, `/rename`, `/archive`, `/theme <key>`,
      `/sessions`, `/help` — grows with milestones.

## E. Streaming feel (lands M0.4, checklist here for the shell pieces)

- [ ] Send button **morphs** to a stop button while streaming (one control,
      two modes); stop aborts cleanly.
- [ ] Auto-scroll pinned to bottom with an **escape hatch**: if the user
      scrolled up > ~300px, streaming does NOT yank them down; sending
      re-pins.
- [ ] New tail tokens fade in (opacity keyframe ~0.4s); finalized markdown
      blocks render once (no whole-message re-render flicker).

## F. Theme system

- [x] The kit's **18 themes** (`putty`, `putty-light`, dark/"Original", light,
      midnight, paper, cyberpunk, retrowave, forest, ocean, ume, copper,
      terminal, organs, lavender, gpt, claude, cute) applied via
      `data-theme` on `<html>`; token-driven (never hardcoded hex).
- [x] Switcher = a theme grid/picker showing swatches; selection persisted
      (localStorage) and applied instantly.
- [x] The **`data-theme-switching` one-frame transition suppression** (from
      the kit) so colors don't stick mid-fade.

## G. Keyboard & Escape discipline

- [x] `Ctrl/Cmd-K` palette · `Ctrl+/` focus composer · `Ctrl+Alt+B` toggle
      sidebar · `Ctrl+Alt+N` new session (once sessions land).
- [x] **Single Escape arbiter**: exactly ONE thing closes per press, priority:
      transient menu → topmost window/overlay → (never steals from typing).
      No per-component Escape handlers that cascade.
- [x] Visible focus states everywhere; the palette is fully keyboard-drivable.

## H. Empty/loading states & motion

- [ ] Loading = spinner rows (kit-styled), not gray skeletons.
- [x] Welcome screen: mascot + greeting + tip when no session is active.
- [x] `prefers-reduced-motion` disables entrance/streaming/window animations.

## Build annotations (2026-07-02) — where the implementation deviates

- **A/sidebar default:** 264px (the kit's `.sb` width), not Odysseus's 340 —
  resizable + persisted either way.
- **B/side-edge behavior:** dragging to a side edge **docks** (reserves chat
  space via CSS var — the modalSnap behavior); the ghost previews the docked
  half-layout, so the "half-screen tile" look is served by the dock rather
  than an overlay tile. Top strip = fullscreen/maximize tiles as specced.
- **C/palette content:** commands + navigation + themes today. No debounce
  needed (local data). Session-grouped results with snippets/timestamps
  arrive with sessions at M0.4+.
- **D/command set today:** `/theme` `/home` `/providers` `/shortcuts`
  `/help` (+ `/settings`, `/keys` aliases). `/new` `/rename` `/archive`
  `/sessions` join at M0.4 (they need sessions). No parent/subcommand pairs
  yet — the machinery supports them when they land. Command echo-as-bubble
  needs the chat transcript (M0.4); today commands execute + toast.
- **E + H/spinner rows:** unticked — they need the M0.4 chat stream and the
  first real loading surfaces.

## Deliberately out of scope for M0.3 (decisions, not omissions)

- Sidebar **side-swap** (shift-click), section **drag-reorder**, the
  email/document split-dock, mobile chip "snake" physics + trash whirlpool,
  route-aware favicon tinting, custom-theme editor + harmony generator,
  AI-driven `ui_control` events, canvas sinewave/whirlpool spinners
  (kit-styled CSS spinner instead), TTS/STT anything (hard rule), remappable
  keybinds (fixed defaults in v1).

## Review protocol

Run PuttyU beside Odysseus (`ODYSSEUS-REF/` running locally) and walk every
checked box side-by-side. A box is checked only when the behavior matches in
feel — timing, thresholds, and persistence included.
